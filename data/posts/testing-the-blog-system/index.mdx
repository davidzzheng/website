## Overview

This post exercises every feature of the blog system: callouts, code groups, steps, live code execution, and component previews. If everything renders correctly here, the blog is working.

## Callouts

<Callout type="info" title="Did you know?">
Astro 7 content collections support MDX components natively. No extra configuration needed — just pass them to the Content renderer.
</Callout>

<Callout type="tip">
Use the `draft: true` frontmatter field to keep posts unpublished in production while still previewing them locally.
</Callout>

<Callout type="warning" title="Watch out">
The `postSlug()` helper strips `/index` from folder-based post IDs. Don't use `post.id` directly in URLs.
</Callout>

<Callout type="danger">
Never use `as any` in Preact components. Import types from `preact`, not `preact/hooks`.
</Callout>

## Code Groups

When you want to show the same concept in multiple languages:

<CodeGroup>

```ts
const greet = (name: string): string => `Hello, ${name}!`
```

```js
const greet = (name) => `Hello, ${name}!`
```

</CodeGroup>

## Steps

<Steps>

### Create the file

Run the scaffold script to create a new post:

```bash
bun run scripts/new-post.ts "My Post" --tag react
```

### Write your content

Open the generated `.mdx` file and start writing. Use MDX components like `<Callout>` and `<Steps>` to enrich your posts.

### Publish

Flip `draft: false` in the frontmatter and your post goes live on the next build.

</Steps>
